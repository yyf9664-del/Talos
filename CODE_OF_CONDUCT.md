# Talos Code of Conduct

## Our Pledge

Talos is an open source project for people who want to build, use, test, and improve local-first AI tooling. We are committed to keeping project spaces constructive, respectful, and useful for contributors and users.

This Code of Conduct applies to all Talos project spaces, including GitHub issues, pull requests, discussions, comments, releases, and any other official community channels.

## Expected Behavior

Examples of behavior that helps the community:

- Focus on the project, the code, the product, and the user problem.
- Be respectful when disagreeing.
- Provide clear reproduction steps, logs, screenshots, or examples when reporting problems.
- Keep feature requests grounded in a real use case.
- Respect maintainers' time and decisions.
- Help keep discussions welcoming for new and experienced contributors.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, insults, threats, personal attacks, or abusive language.
- Spam, advertising, phishing, scams, or unrelated promotion.
- Automated low-quality issues, comments, pull requests, or reactions.
- Repeatedly posting the same or substantially similar content.
- Derailing technical discussions with unrelated arguments.
- Publishing private information without explicit permission.
- Reporting security vulnerabilities publicly instead of following the security policy.
- Any conduct that maintainers reasonably determine to be harmful to the project or community.

## Enforcement

Maintainers may take any action they consider appropriate to protect the project and community, including:

- Asking a participant to clarify, edit, or stop a behavior.
- Closing issues or pull requests that are off-topic, unactionable, abusive, spam, or automated low-quality submissions.
- Hiding, editing, or deleting comments.
- Locking conversations.
- Applying GitHub interaction limits.
- Reporting abuse or spam to GitHub.
- Temporarily or permanently blocking participation in project spaces when available.

Maintainers are not required to debate moderation decisions in public threads, especially during spam, harassment, or coordinated abuse.

## Reporting

If you see behavior that violates this Code of Conduct, report it to the maintainers through GitHub's reporting tools where available.

Security vulnerabilities must not be reported in public issues. Please follow the process in [SECURITY.md](SECURITY.md).

## Scope

This Code of Conduct applies within Talos project spaces. It may also apply outside project spaces when behavior directly targets Talos contributors, maintainers, users, or the health of the project community.
