# Domain Docs

This is a single-context repo.

Before architecture, diagnosis, TDD, or issue-writing work, read:

- `CONTEXT.md` at the repo root for project vocabulary
- Relevant ADRs under `docs/adr/`
- `.dev/ARCHITECTURE.md` when module dependency rules matter
- `.dev/CONVENTIONS.md` when coding or testing conventions matter

Use the glossary terms from `CONTEXT.md` in issue titles, refactor proposals, hypotheses, and tests.

If a proposal contradicts an ADR, call that out explicitly instead of silently overriding it.
